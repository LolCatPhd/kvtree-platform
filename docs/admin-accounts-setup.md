# KV Tree — Accounts to create

Hi! 👋 To run the KV Tree website, we need a few free online accounts in **your**
name, so you're the owner of everything (the code, the website, and the
database). Right now they're under my account — this moves them home to you.

You can do all of this **on your phone**. Where a page is fiddly on mobile, tap
**"Desktop site"** (Safari: **aA** → *Request Desktop Website*; Chrome: **⋮** →
tick *Desktop site*).

**There are 3 accounts**, and they all link together:

| # | Account | What it's for |
|---|---------|---------------|
| 1 | **GitHub** | Stores the website's code (the "master copy"). |
| 2 | **Netlify** | Runs the public **website** (what customers see). |
| 3 | **Railway** | Runs the **back office** — the login system, CRM, and database. |

👉 **Do GitHub first** — because you'll log into Netlify and Railway *using* your
GitHub account (one login for all three). 

**The plan:** once these exist, you send me your **GitHub username**, I move the

## Account 1 — GitHub (do this first)

GitHub is where the website's code lives.

1. In your phone browser go to **https://github.com/signup**
2. Enter:
   - **Email:** leslie.strydom@gmail.com
   - A **password** (save it somewhere safe).
   - A **username** — e.g. `kvtree` or `leslie-kvtree`. ✍️ **Remember this — you
     send it to me.**
3. Solve the little puzzle, then **verify your email** (GitHub sends a code).
4. If it asks about plans, choose the **Free** plan. Skip any "team"/paid
   upsell.
5. ✅ Done. **Send me your GitHub username** so I can move the code over and add
   you as the owner.

> 🔐 **Recommended:** turn on two-factor authentication later
> (Settings → Password and authentication). I can walk you through it — it keeps
> the code safe.

## Account 2 — Netlify (sign in with GitHub)

Netlify runs the public website customers visit.

1. Go to **https://app.netlify.com/signup**
2. Tap **"Sign up with GitHub"** (the GitHub button — **not** email).
3. GitHub asks **"Authorize Netlify?"** → tap **Authorize**.
4. If Netlify asks what to do next, you can **stop here** — just being signed up
   is enough. I'll connect the website to it.
5. Choose the **Free / Starter** plan if prompted. No card needed.
6. ✅ Done.

---

## Account 3 — Railway (sign in with GitHub)

Railway runs the back office (logins, CRM, database).

1. Go to **https://railway.com** (or **https://railway.app**) → **Login**.
2. Tap **"Login with GitHub"**.
3. GitHub asks to **Authorize Railway** → tap **Authorize**.
4. You may be asked to verify your account (a quick email/GitHub check).
5. You can **stop here** — I'll create the project and database inside it.
6. ✅ Done.
